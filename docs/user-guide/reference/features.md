# Sqush single-image editor — feature reference

Exhaustive, code-derived inventory of every feature and user interaction in the
single-image editor. Each entry lists the source file and one precise sentence
on what it does. Derived purely from reading the source — see "Notable / hidden
behaviors" at the end for things not surfaced in the obvious UI.

| Orientation (horizontal/vertical) | `Output.svelte` (`orientation` derived), `two-up.css`                          | The split is vertical (top/bottom) when the viewport is ≤760px wide and horizontal (left/right) otherwise; the handle, clip-paths, and scrubber rotation all switch accordingly.                                                                     |

| Synced pan/zoom across sides      | `Output.svelte` (`onLeftChange`), `output/pinch-zoom.ts`                       | Only the left `<pinch-zoom>` is driven; its `change` event mirrors scale/x/y onto the right `<pinch-zoom>` so both halves stay perfectly aligned.                                                                                                    |
| Event retargeting to left side    | `Output.svelte` ($effect with capture listeners)                               | Touch/mouse/wheel (+ pointerdown on non-Safari) events on the two-up are cloned and re-dispatched to the left pinch-zoom; drags over the handle pass through (so the slider works) but wheel always retargets so zooming works even over the handle. |
| Pinch-zoom (touch)                | `output/pinch-zoom.ts` (`_onPointerMove`)                                      | Two-finger pinch scales about the gesture midpoint and pans by the midpoint delta; one-finger drag pans the image.                                                                                                                                   |
| Auto fit + centre on load/resize  | `Output.svelte` ($effect with `fittedKey`)                                     | On a new file, dimension change, orientation flip, or viewport resize (keyed, so re-encodes at the same size don't refit), the image is scaled to fit (never above 1×) and centred inside the inset-adjusted viewport via a `requestAnimationFrame`. |
| Fit insets                        | `Output.svelte`, `src/routes/+page.svelte` (`--fit-inset-*`)                   | Desktop reserves 300px on each side (`--fit-inset-left/right`) so the fitted image clears the two option panels; on mobile (≤760px) these insets collapse to 0.                                                                                      |
| Contain-letterbox alignment       | `Output.svelte` (`containStyle`), `editor-session.svelte.ts` (`sideContains`)  | When a side uses Resize with fit-method "contain", its canvas CSS box is pinned to the original source dimensions with `object-fit:contain`, so a smaller output stays letterboxed inside the original footprint and the two halves remain aligned.  |

## Notable / hidden / non-obvious behaviors

- **Keyboard split presets (1/2/3)** in `two-up.ts` jump the comparison slider to start/middle/centre but are undocumented in the UI and suppressed while typing in an input.
- **Wheel always retargets to the left pinch-zoom**, so scroll-to-zoom works even with the cursor over the drag handle (`Output.svelte` event retargeting).
- **Safari-specific paths**: the smoothing/pixelated toggle is omitted entirely on Safari (`Output.svelte`); `pointerdown` is excluded from the retargeted event list on Safari; and `pinch-zoom.ts` sets `avoidPointerEvents: isSafari` to work around an iOS WebKit pointer-capture bug (#220196).
- **Android keyboard blur**: on `touchend` over the output, the active element is blurred so Android Chrome doesn't re-pop the software keyboard (ported `shouldBlurActiveElementAfterOutputRetarget`).
- **Firefox focus workaround**: the editable zoom input forces a style recalc (`getComputedStyle().transform`) before `focus()`/`select()` so Firefox doesn't drop focus on the just-inserted input (`focusOnMount`).
- **Quantize "ZX" konami easter-egg is intentionally OMITTED** from the Svelte port — `QuantizeOptions.svelte` notes the original's hidden konami-code palette type was dropped.
- **Trackpad pinch vs wheel**: `pinch-zoom.ts` distinguishes trackpad pinch (`ctrlKey`) from a plain wheel by using a tighter zoom divisor, and scales "lines"-mode deltas ×15.
- **Inverted-value sliders**: several "Effort"/"Slight loss"/"Filter sharpness" sliders display the inverse of the stored codec value (e.g. WebP near_lossless, filter_sharpness; AVIF effort=10−speed), so the displayed scale is friendlier than the raw option.
- **Lossless inference**: AVIF/JXL panels infer the Lossless checkbox state from the underlying quality/subsample rather than a dedicated stored flag, and JXL force-enables "Alternative lossy mode" (disabled) below quality 7.
- **Save vs global persistence asymmetry**: the global auto-persist (`sqush:settings:v2`) stores only format + options (not processor/resize/quantize state), whereas the per-side Save button (`sqush:side-settings:*`) stores the full state including processor state — so a reload restores encoder choices but not resize/quantize toggles.
- **Import validation gating**: the Import button's enabled state and the import action share one `parseSavedSide` validator, so a corrupt or partial saved payload both disables the button and (if forced) shows an "invalid" snackbar — they can't drift.
- **`legacy-clip-compat` mode**: `<two-up>` positions its children absolutely and uses `clip-path` insets (rather than transform-based clipping) for cross-browser compatibility.
- **Identity/Original download is the raw file**: the Original side's download is `URL.createObjectURL(file)` of the untouched original (0% change), not a re-encode.
- **No-JS / SSR safety**: `localStorage` access is guarded (`canUseLocalStorage`) so private-browsing/SSR don't throw; settings simply don't persist there.
