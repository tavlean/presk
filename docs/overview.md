# Presk Project Overview

Presk is a browser-local image optimizer. Users pick or drop an image, the app
decodes it in the browser, lets the user compare output settings, and downloads
the optimized file from an object URL. There is no backend image-processing
service.

## What Runs Where

- SvelteKit renders the client-only app shell.
- Svelte 5 components and `.svelte.ts` modules own the editor UI and reactive
  state.
- Framework-neutral TypeScript helpers decode, preprocess, resize, quantize, and
  encode images.
- Web Workers run codec and processor work so the UI stays responsive.
- WebAssembly codecs live under `codecs/`.
- The SvelteKit service worker caches the app shell and generated codec assets
  for offline reload.

## Main Folders

- `src/routes/`: SvelteKit routes.
- `src/lib/`: Svelte editor, worker bridge, compression adapter, diagnostics,
  and service-worker registration.
- `src/client/lazy-app/`: framework-neutral image pipeline, browser decode
  helpers, object filename helpers, worker runtime helpers, and bulk helpers.
- `src/features/`: encoder/decoder/processor/preprocessor metadata and runtimes.
- `src/shared/`: framework-neutral shared contracts.
- `src/sw/`: cache-plan helpers and service-worker fixtures.
- `src/worker-shared/`: worker support shared by codecs.
- `codecs/`: committed codec JS/WASM artifacts.
- `scripts/`: SvelteKit generator and build-output audit.
- `static/`: logo and install icons.
- `build/`: generated production output from `npm run build`.

## Image Flow

1. `src/routes/+page.svelte` accepts a file.
2. `src/lib/editor/editor-session.svelte.ts` owns side state, saved settings,
   undo/redo history, a shared result cache (with object-URL cleanup), and encode
   scheduling.
3. `src/lib/compress.ts` decodes, preprocesses, processes, and encodes through
   the shared image pipeline.
4. `src/lib/sveltekit-worker-bridge.ts` calls the generated worker surface for
   codec and processor work.
5. `src/lib/editor/output/Output.svelte` renders the before/after comparison.
6. The result is downloaded from a browser object URL.

## Feature Wiring

Feature metadata and runtime wrappers are generated by:

```sh
npm run sync
```

The generator writes `.svelte-kit/app-generated/*`:

- `feature-meta/*` for encoder/processor/preprocessor metadata;
- `features-worker/webp.ts` for the generated SvelteKit worker entry;
- `worker-bridge/meta.ts` and `worker-surface/ready.ts`;
- `codec-assets/*` for Vite `?url` WASM imports and precache records;
- patched codec wrapper copies under `codecs/*`.

These files are generated output and are not committed.

## Deployment Notes

The app builds with SvelteKit `adapter-static` and a `200.html` fallback. The
service worker registers from app code so development can unregister stale
preview workers and production can cache the exact generated assets.

Threaded codec paths are future performance work. The launch baseline is
single-thread codec parity.
